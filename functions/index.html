Functions Quick Reference

Go templates are lightweight but extensible. Go itself supplies built-in functions, including comparison operators and other basic tools. These are listed in the Go template documentation. Stencil exposes more functions through Sprig and some itself.

append

append appends one or more values to a slice and returns the resulting slice. Read More »

base64

base64Encode and base64Decode let you easily decode content with a base64 encoding and vice versa through pipes. Read More »

default

Allows setting a default value that can be returned if a first value is not set. Read More »

dict

Creates a dictionary from a list of key and value pairs. Read More »

eq

Returns the boolean truth of arg1 == arg2. Read More »

file.Block

Block returns the contents of a given block ###Block(name) Hello, world! ###EndBlock(name) Read More »

file.Create

Create creates a new file that is rendered by the current template. If the template has a single file with no contents this file replaces it. Read More »

file.Delete

Delete deletes the current file Read More »

file.Path

Path returns the current path of the file we’re writing to. Read More »

file.SetContents

SetContents sets the contents of the current file to the provided string. Read More »

file.SetPath

SetPath changes the path of the current file Read More »

file.Skip

Skip skips the current file Read More »

file.Static

Static marks the current file as static Read More »

ge

Returns the boolean truth of arg1 >= arg2. Read More »

le

Returns the boolean truth of arg1 <= arg2. Read More »

len

Returns the length of a variable according to its type. Read More »

lower

Converts all characters in the provided string to lowercase. Read More »

lt

Returns the boolean truth of arg1 < arg2. Read More »

ne

Returns the boolean truth of arg1 != arg2. Read More »

now

Returns the current local time Read More »

stencil.AddToModuleHook

AddToModuleHook adds to a hook in another module Read More »

stencil.ApplyTemplate

ApplyTemplate executes a template inside of the current module that belongs to the actively rendered template. It does not support rendering a template from another module. Read More »

stencil.Arg

Arg returns the value of an argument in the service’s manifest. Read More »

stencil.Args

Args returns all arguments passed to stencil from the service’s manifest. Note: This doesn’t set default values and is instead representative of all data passed in its raw form. Read More »

stencil.GetModuleHook

GetModuleHook returns a module block in the scope of this module Read More »

with

Rebinds the context (.) within its scope and skips the block if the variable is absent or empty. Read More »
